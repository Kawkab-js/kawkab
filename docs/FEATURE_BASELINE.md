# Feature baseline

This file is the **contractual baseline** for what the repository aims to ship. When behavior changes materially, update this document in the same change as the code and refresh `docs/NODE_COMPATIBILITY.md` where relevant. Status semantics: `[docs/COMPAT_DEFINITION_OF_DONE.md](COMPAT_DEFINITION_OF_DONE.md)`. Product non-goals: `[docs/NODE_NON_GOALS.md](NODE_NON_GOALS.md)`. **Ecosystem compatibility targets** (Express, NestJS, Prisma, Next custom server, Top 100 basket): `[docs/COMPAT_KPI.md](COMPAT_KPI.md)`. Reference scenarios: `[docs/NPM_CORPUS.md](NPM_CORPUS.md)`.

## Workspace layout

- `**kawkab-core`** (directory `core/`): QuickJS-backed runtime, Node-style bootstrap, module loading, transpilation (SWC), bytecode cache, HTTP/IO integration surface.
- `**bridge**`: Console bridge install hook and flush helpers for the CLI path.
- `**io**`: I/O primitives; optional `**tokio-uring**` feature for Linux.
- `**snapshot**`: Experimental snapshot manifest metadata (not a full VM snapshot image).
- `**kawkab-cli**` (directory `kawkab/`): `kawkab` binary — script execution, engine modes (`auto` / QuickJS / Node fallback), package-manager subcommands.
- `**pm**`: Native package manager library used by the CLI (`package.json`, lockfile, registry install, `kawkab run`, `why`, `doctor`).

## Build and platform

- Primary target: **Linux** (including WSL2). Workspace builds with `--features tokio-uring` on supported kernels.
- QuickJS via `**hirofa-quickjs-sys`** (workspace version); compatibility layer in `core/src/qjs_compat.rs`.

## Runtime capabilities (summary)

- Execute JS/TS/JSX/TSX via transpilation; CommonJS `require` and ESM entrypoints as documented in `docs/NODE_COMPATIBILITY.md`. Module resolution includes bare-specifier package/subpath splitting, `package.json` `exports` / `imports` (conditional and pattern subset), `require('module').createRequire`, and `**process.env.NODE_ENV**` driving `development` / `production` export conditions during resolution.
- Expanded built-in surface (still best-effort vs Node): `fs` (`copyFileSync`, `rmSync`), `path` (`relative`, `parse`), `zlib` (`deflateSync`, `inflateSync`), `punycode` IDNA (`toASCII` / `toUnicode` via `idna`), `crypto` hashes (`md5`, `blake3`), `http` / `https` blocking client (`get`, `request` via reqwest; servers remain simplified; HTTPS server is not TLS-terminated in-process).
- Runtime compatibility uplift includes: `async_hooks` compatibility baseline (`AsyncLocalStorage` with `run`/`enterWith`/`getStore`/`exit`/`bind`, `AsyncResource`), improved `events` module helpers (`once` Promise + `on` async iterator baseline), richer `structuredClone` polyfill (`Date`/`Map`/`Set`/typed arrays), functional `vm` baseline (`Script`, `createContext`, `isContext`, `runInContext`), `tls` compatibility shim with socket/server shape contracts (`connect`, `createServer`, `secureConnect` event), upgraded `dns` host-resolution path for `lookup`/`resolve4`/`resolve6`, stable `dgram` socket baseline (`createSocket`, listener helpers, `bind`/`send`/`close`/`address`), stable `punycode` domain conversion path (`toASCII`/`toUnicode` via IDNA) plus ASCII `encode`/`decode` baseline, stable global base64 helpers (`atob`/`btoa`) for common encode/decode flows, stable Web API HTTP surface (`fetch`, `Headers`, `Request`, `Response`) and text codec constructors (`TextEncoder`, `TextDecoder`) plus text transform streams (`TextEncoderStream`, `TextDecoderStream`), stable queue strategy globals (`ByteLengthQueuingStrategy`, `CountQueuingStrategy`), stable Web Streams baseline (`ReadableStream`, `WritableStream`, `TransformStream`) and compression stream constructors (`CompressionStream`, `DecompressionStream`), stable stream placeholder constructors for feature detection (`Readable*`/`Writable*`/`TransformStreamDefaultController` illegal-constructor behavior), stable DOM-style event globals (`Event`, `EventTarget`, `CustomEvent`), stable messaging globals (`MessageEvent`, `MessagePort`, `MessageChannel`) plus in-process `BroadcastChannel`, stable `Atomics` shim for deterministic typed-array atomic-style operations (without blocking wait semantics), stable WebCrypto baseline (`crypto.getRandomValues`, `crypto.randomUUID`, `crypto.subtle.digest`) with `CryptoKey`/`SubtleCrypto` global illegal-constructor placeholders, stable WebAssembly compatibility baseline (`Module`, `Instance`, `Memory`, `Table`, `Global`, `validate`, `compile`, `instantiate`) for detection/control-flow paths, stable Node compatibility stubs for non-core-heavy modules (`node:tty`, `node:domain`, `node:repl`, `node:trace_events`, `node:inspector`, `node:v8`, `node:wasi`, `node:cluster`, `node:http2`, `node:sqlite`) to satisfy feature detection and guarded package init flows, stable `DOMException`/`FormData` constructor+method baseline (`name`/`message`/`code`; `append`/`delete`/`get`/`getAll`/`has`/`set`), stable `Blob` baseline (`size`/`type`/`slice`/`arrayBuffer`/`text`), stable `URL`/`URLSearchParams` query/url helpers (`URL` fields + live `searchParams` reflected in `href`, `append`/`set`/`get`/`getAll`/`delete`/`toString`), improved non-interactive `readline` baseline plus `readline/promises` Promise `question()` path, `module.createRequire` rooted loading contract, sync `zlib` roundtrip contract (`gzipSync`/`gunzipSync`), and nested `worker_threads` constructor baseline in worker isolates.
- **worker_threads:** one OS thread per worker with an isolated QuickJS runtime; `postMessage` / `parentPort` payloads are **JSON-serializable** only in this baseline (see **Remaining vs Node** in `NODE_COMPATIBILITY.md`). Worker `parentPort` keeps an ordered **multi-listener** queue with stable per-listener ids for `once` teardown (`core/src/node/worker_threads.rs`). Harness contracts that spawn OS workers use `WORKER_THREADS_OS_HARNESS_SERIAL` in `compat_contract_tests.rs` to reduce cross-test interference under parallel `cargo test`. Active gates include `worker_parent_port_remove_all_listeners_contract` (main→worker `ping` arm; host install `/test/main.js` + `drain_next_tick_queue` in harness) and `worker_parent_port_once_one_shot_contract` (warmup + delayed `{kind:'once'}` after `parentPort.once`; no worker-side `setInterval`; host `/test/main.js`). `worker_a_receive_message_on_port_baseline_contract` is the `receiveMessageOnPort` / `MessageChannel` baseline (name sorts before other `worker_*` tests so `cargo test … worker` runs it before OS `Worker` harnesses and avoids cumulative `SIGABRT`). All OS-`Worker` harnesses in this module take `WORKER_THREADS_OS_HARNESS_SERIAL` plus `qjs_serial()`.
- `**Isolate::eval`** passes an exact byte length for the `CString` script buffer (excluding the implicit NUL) to match QuickJS `JS_Eval` expectations and avoid out-of-bounds reads.
- Priority built-in contract: `node::compat_contract_tests::priority_and_web_platform_embed_contract` in `core/src/node/compat_contract_tests.rs` (requires network for `example.com` HTTP/HTTPS checks).
- Web platform / streams / messaging baseline: `Blob`, `FormData`, `Event`/`EventTarget`/`CustomEvent`, `DOMException`, WHATWG-style `**ReadableStream**` / `**WritableStream**` / `**TransformStream**` (queue baseline), `**CompressionStream**` / `**DecompressionStream**` (`gzip`/`deflate`, full-buffer flush via `flate2`), `**MessageChannel**` / `**MessagePort**` / `**BroadcastChannel**` (in-process; synchronous delivery in the same isolate), plus expanded `**os**` (`EOL`, `loadavg`, `networkInterfaces` stub), `**querystring**` (`escape`/`unescape`, `parse`/`stringify` options), `**timers/promises.setInterval**` (async iterable), `**string_decoder**` (UTF-8/UTF-16LE carry + Buffer input), `**readline**` stub surface, `**util.types**` (`isNull`/`isUndefined`/`isRegExp`/`isBuffer`), `**perf_hooks**` (`PerformanceMark`/`PerformanceEntry`/… stubs and `globalThis` performance ctors), and `**node:test**` `before`/`after`/`beforeEach`/`afterEach`. Contract: `node::compat_contract_tests::priority_and_web_platform_embed_contract` (`core/src/node/web_platform_shim.rs` + `web_platform_shim.js`, installed after `Buffer`).
- Phase-1 behavioral parity contracts added for runtime ordering and worker lifecycle: `event_loop_ordering_contract` and `worker_threads_lifecycle_contract`.
- Additional high-risk behavior contracts: `stream_pipeline_backpressure_contract` (pipeline drain/error handler path). `http_client_local_behavior_contract` exists but remains ignored pending deeper FFI/runtime hardening.
- Repeatable checks: `RUST_TEST_THREADS=1 cargo test --workspace --features tokio-uring` (CI sets `RUST_TEST_THREADS=1` on the workflow so workspace test binaries do not run in parallel; avoids `worker_threads` / QuickJS harness `SIGABRT` under overlap). `[scripts/compat_smoke.sh](../scripts/compat_smoke.sh)` exports the same default, strips inherited `LD_LIBRARY_PATH` entries under `target/*/build/*/out` (Cargo re-injects its own for children), runs `cargo test --workspace --features tokio-uring --lib` (library tests only, matching stable harness flags), then re-invokes `worker_a_receive_message_on_port_baseline_contract`, the four core `worker_threads_*` harnesses, and `worker_parent_port_*` contracts as explicit `cargo test -p kawkab-core …` gates, each with `--test-threads=1 --nocapture` (avoids QuickJS `SIGABRT` under libtest stdout capture on some Linux setups). `install_runtime` clears in-process `MessageChannel` port peer/queue/ref registries between isolates on the same OS thread (`worker_threads::clear_install_thread_js_handles`). `[scripts/kpi_smoke.sh](../scripts/kpi_smoke.sh)` (`http|express|all`), and performance gate `[scripts/runtime_perf_gate.sh](../scripts/runtime_perf_gate.sh)`.
- Node compatibility is **best-effort**; built-ins and globals are a curated subset — see the compatibility matrix, not npm “works everywhere”. **Directed** compatibility goals (percent targets per stack) live in `[COMPAT_KPI.md](COMPAT_KPI.md)`; the frozen Top 100 package basket is [`data/top100-packages.txt`](data/top100-packages.txt).
- **Security:** `child_process` and similar host capabilities are policy-gated (e.g. `KAWKAB_ALLOW_CHILD_PROCESS`); default is restrictive.

## Package manager (CLI)

- Reads/writes `package.json` dependency sections; maintains `kawkab.lock` with resolved URLs and integrity.
- Registry fetch and tarball cache under OS cache (`.../kawkab/packages`).
- Workspace selectors: `workspace:`*, `workspace:^`, `workspace:~`.
- Diagnostics: `kawkab why` and `kawkab doctor` support human and JSON output; schema/fixture policy in `pm/fixtures/README.md`.

## Native vector API

- `kawkab.vec.*` typed-array helpers (Rust offload); legacy `kawkab.fast*` retained for compatibility — see root `README.md`.
